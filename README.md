# Walmart Sales Analysis

## 📌 Project Overview
This project delivers an end-to-end analysis of Walmart sales data to uncover actionable insights into customer behavior, sales performance, and operational trends. Using MySQL for data analysis and Power BI for visualization, the project transforms raw transactional data into meaningful business intelligence.

---

## 🎯 Business Objectives
- Analyze customer payment preferences and transaction patterns  
- Identify top-performing product categories across branches  
- Determine peak sales days and high-traffic periods  
- Evaluate sales distribution across different time shifts  
- Assess category-level profitability  
- Detect branches with declining revenue trends  

---

## 🛠️ Tools & Technologies
- MySQL – Data extraction, transformation, and analysis  
- Power BI – Interactive dashboard and data visualization  
- Excel/CSV – Data source  

---

## 📊 Key Insights
- Customer payment behavior varies significantly across branches  
- Certain product categories consistently receive higher ratings  
- Sales volume peaks on specific days and time periods  
- High-profit categories contribute disproportionately to revenue  
- Revenue decline in select branches indicates potential operational gaps  

---

## 📈 Dashboard Highlights
- KPI Metrics: Total Revenue, Profit, Transactions, Average Rating  
- Sales trend analysis over time  
- Branch and city-level performance comparison  
- Category-wise revenue and profitability insights  
- Dynamic filtering using slicers for interactive exploration  

---

## 🚀 Conclusion
This project demonstrates the ability to apply SQL and data visualization tools to solve real-world business problems, enabling data-driven decision-making in a retail environment.
